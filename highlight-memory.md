# AI Memory Gateway

**项目描述**：开发了一个轻量级 LLM 代理网关，为 AI 对话系统添加长期记忆层，支持任意 OpenAI 兼容客户端与多模型服务商。

**技术栈**：Python / FastAPI / PostgreSQL / asyncpg / httpx / jieba

---

## 亮点 1：三层记忆架构

**问题**：传统 AI 对话无持久化记忆，每轮对话都是独立的上下文。

**方案**：设计并实现「碎片→事件→核心」三层记忆分层架构：
- L1 碎片：自动从对话中提取原始记忆
- L2 事件：AI 自动整理合并为完整事件
- L3 核心：手动标记的重要记忆优先注入

**成果**：用户可根据重要程度灵活管理记忆注入，核心记忆优先触达。

---

## 亮点 2：分区缓存 + Prompt Caching 优化

**问题**：长对话上下文 token 成本高昂。

**方案**：
- 设计 A/B 区轮转 + 摘要压缩机制，每 15 轮自动触发摘要压缩
- 利用 Prompt Caching，正常轮次 90% 历史消息走缓存读取

**成果**：
- 缓存命中时 token 成本降低至 **0.1x**
- 缓存命中率达 **90%**
- 支持时间窗口触发模式，适配高频消息场景（如微信）

---

## 亮点 3：四维混合向量搜索

**问题**：纯关键词搜索无法理解语义（如"过年"搜不到"春节"）。

**方案**：
- 集成 OpenAI 兼容 Embedding API（支持 Jina/Voyage/Ollama）
- 实现四维加权评分：关键词(0.35) + 语义相似度(0.35) + 重要程度(0.15) + 时间衰减(0.15)
- 支持 pgvector 自动检测，数据库无 pgvector 时回退到 Python 端余弦相似度计算

**成果**：语义搜索成功率大幅提升，关键词 + 语义双路检索互补。

---

## 亮点 4：工程细节

- **兼容性**：自动检测非 Claude 模型，发送前剥离 `cache_control` 字段，解决智谱 GLM 等模型兼容问题
- **容错设计**：JSON 解析三层容错（strict → 去控制字符 → AI 修复），适配模型非标准输出
- **热更新**：运行时配置可在 Dashboard 在线修改，配置变更即时生效无需重启
- **部署设计**：零成本起步，可部署在 Render 免费额度内，支持 Docker 容器化
