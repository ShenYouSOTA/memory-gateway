## 分层职责

### Routes 层
- HTTP 请求/响应处理
- 参数验证
- 调用 Service 层

### Services 层
- 业务逻辑封装
- 跨 Repository 协调
- 不关心存储实现

### Repositories 层
- 数据访问抽象
- 定义接口（ABC）
- PostgreSQL/zvec 实现

## 数据流

### 记忆搜索流程
```
1. 客户端发送查询
2. Route 接收参数
3. Service 调用 MemoryRepo.search()
4. MemoryRepo 执行混合搜索：
   - 关键词路：PostgreSQL ILIKE
   - 向量路：zvec 向量检索
   - 四维加权排序
5. 返回结果给客户端
```

### 记忆提取流程
```
1. 对话结束触发
2. ExtractionService 调用 LLM
3. LLM 返回新记忆
4. MemoryService 保存：
   - MemoryRepo.save() → PostgreSQL
   - VectorRepo.insert() → zvec
```

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 架构模式 | 分层架构 | 职责清晰，易于测试 |
| 存储抽象 | Repository 模式 | 可切换实现 |
| 向量数据库 | zvec（替代 pgvector） | 嵌入式、高性能、易部署 |
| 依赖注入 | FastAPI Depends | 松耦合、可测试 |
